# Family Weekly Menu

A static web app for browsing weekly family menus, recipes, shopping lists, and nutrition validation. Hosted on GitHub Pages. No backend, no build step.

## Local development

`fetch()` requires HTTP — opening `index.html` directly from the filesystem will fail. Use a local server:

```bash
python3 -m http.server 8080
# then open http://localhost:8080
```

Or use VS Code Live Server extension.

## Weekly workflow (manager)

1. Generate a week JSON using the LLM prompt in `prompts/weekly-menu-generation-prompt.md`
2. Save the output as `data/weeks/YYYY-Www.json` (e.g. `2026-W20.json`)
3. Run the sync script to update the manifest:
   ```bash
   node scripts/sync-weeks-index.js
   ```
4. Validate the new week:
   ```bash
   node scripts/validate-week.js data/weeks/2026-W20.json
   ```
5. Commit and push — GitHub Pages updates automatically

Full workflow details and troubleshooting: `docs/OPERATIONS.md`

## How the week dropdown works

The app reads `data/weeks/index.json` (auto-generated by the sync script). The default week is selected in this priority order:

1. `?week=` URL parameter
2. Previously selected week (localStorage)
3. `defaultWeekId` from the manifest
4. Current week (today falls between start and end date)
5. Nearest upcoming week
6. Most recent past week

## Shopping list

Checkboxes persist per-week in localStorage. Switching to a different week shows a fresh unchecked list. Use the **Reset all** button to uncheck everything for the current week.

## Recipe favorites

The star button on each recipe card saves favorites to localStorage. They persist across sessions and week changes. Use the **Favorites only** toggle to filter the recipe list.

## Nutrition estimates

All calorie and macro figures are LLM estimates, not precise nutritional analysis. Use them for rough planning only.

## GitHub Pages setup

1. Go to repository Settings → Pages
2. Source: Deploy from a branch
3. Branch: `main`, folder: `/ (root)`
4. Save — the site will be live at `https://<username>.github.io/<repo>/`

## Repository structure

```
data/weeks/       Weekly JSON files + auto-generated index.json
prompts/          LLM generation prompt, schema reference, family context
scripts/          Node.js sync and validation scripts (no dependencies)
docs/             Technical documentation
index.html        App entry point
app.js            All frontend logic
styles.css        All styles (mobile-first)
```
